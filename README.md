# 2026年下半期 星の流れビジュアライザー

2026年6月から12月までの天体移動データを、クライアントの出生天体とハウスカスプに重ねて表示する静的Webアプリです。

このアプリは占星術の自動計算アプリではありません。`data.js` に入力済みの `transitData` と `clientSampleData` を読み込み、ホロスコープ円・月別一覧・新月満月一覧として可視化します。

## 使い方

1. `index.html` をブラウザで開きます。
2. クライアント名、出生天体、1ハウスから12ハウスまでのカスプを入力します。
3. 2026年6月から12月の月ボタンを押すと、その月のトランジット表示に切り替わります。
4. ホロスコープ円には、出生天体、トランジット開始位置、終了位置、移動線、逆行マーク、新月満月が表示されます。
5. 月別データ一覧には、開始位置・終了位置・星座移動日・逆行順行情報・ハウス判定が表示されます。

## 入力方法

出生天体とハウスカスプは、星座プルダウンと度数入力で指定します。

度数は各星座内の整数度数です。たとえば双子座11度は、360度上では71度として扱われます。

出生時間が不明な場合は、ASC、MC、ハウスカスプを「不明」にできます。その場合、ハウス判定は「不明」として表示されます。

## 保存方法

入力内容はブラウザの `localStorage` に保存できます。

- `保存`: 現在の入力内容を保存します。
- `読み込み`: 保存済みデータを読み込みます。
- `全データを書き出し`: 保存済みクライアント一覧を `starflow-clients.json` として書き出します。
- `JSONから読み込み`: 書き出したJSONファイルを読み込みます。同名クライアントは上書きし、未登録のクライアントは追加します。
- `サンプルデータを入れる`: `data.js` の `clientSampleData` をフォームに入れます。
- `入力クリア`: 入力内容を初期状態に戻します。

Dropboxなどの同期フォルダに `starflow-clients.json` を保存すると、PCとスマホの間で手動移行できます。クライアントデータはGitHub Pagesやリポジトリには保存されません。

この環境では、書き出したJSONファイルはDropboxの `星よみ2026下半期データー` フォルダに保存する運用を想定しています。

## GitHub Pagesで公開する方法

1. このフォルダの `index.html`、`styles.css`、`app.js`、`data.js`、`README.md` をGitHubリポジトリに置きます。
2. GitHubのリポジトリ設定から `Pages` を開きます。
3. Sourceで公開したいブランチを選び、フォルダは `/root` を選択します。
4. 保存後、表示されたGitHub PagesのURLにアクセスします。

ビルドツール、APIキー、外部API、サーバー処理は不要です。

## 注意事項

このアプリは天体位置を自動計算しません。表示される天体移動、新月満月、逆行順行などの情報は、すべて `data.js` に入力されたデータをもとにしています。

実運用では、公開前に `data.js` の `transitData` を確認済みのデータへ差し替えてください。
